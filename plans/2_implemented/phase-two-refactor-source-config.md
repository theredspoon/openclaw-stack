# Restore source-config.sh with stack.env Architecture

## Context

`deploy/scripts/source-config.sh` was accidentally deleted in commit `2372c30` (Phase 2 cleanup). It was the **single config resolver** sourced by all 17 local scripts, 2 lib files, and 4 deploy scripts. Without it, every script in `scripts/` is broken. This plan restores it with a new architecture that reads from `.deploy/stack.env` (generated by pre-deploy) instead of the old `openclaw-config.env`.

**Design decisions (confirmed with user):**

- Rename script variables to match `.env` names (no alias layer)
- Pre-deploy generates `.deploy/stack.env` alongside `.deploy/stack.json`
- `stack.env` is the **single source** for all shell scripts — never source `.env` directly
- Var naming mirrors source structure: `ENV__<key>` for .env vars, `STACK__<path>` for stack.yml vars (double underscore = path separator)
- Only emit vars that are currently needed by scripts (add as needed, not proactively)

## Step 2: Restore deploy/scripts/source-config.sh

**File:** `deploy/scripts/source-config.sh` (new version)

```bash
#!/usr/bin/env bash
# source-config.sh — Config resolver for OpenClaw scripts.
# Sources stack.env (generated by pre-deploy) for all config variables.
# Auto-detects local (dev machine) vs VPS context.
#
# Exports ENV__* vars (from .env) and STACK__* vars (from stack.yml).
# Also exports: OPENCLAW_CONTEXT ("local"|"vps"), REPO_ROOT (local only)

_SELF_DIR="$(cd "$(dirname "${BASH_SOURCE[0]:-$0}")" && pwd)"
_DEPLOY_DIR="$(cd "$_SELF_DIR/.." && pwd)"
_CANDIDATE_ROOT="$(cd "$_DEPLOY_DIR/.." 2>/dev/null && pwd)"

if [ -f "${_CANDIDATE_ROOT}/.env" ]; then
  OPENCLAW_CONTEXT="local"
  REPO_ROOT="$_CANDIDATE_ROOT"
  _STACK_ENV="${REPO_ROOT}/.deploy/stack.env"
else
  OPENCLAW_CONTEXT="vps"
  REPO_ROOT=""
  _STACK_ENV="${_DEPLOY_DIR}/stack.env"
fi

if [ ! -f "$_STACK_ENV" ]; then
  echo "Error: stack.env not found at $_STACK_ENV" >&2
  echo "Run 'bun run pre-deploy' to generate it." >&2
  exit 1
fi

set -a
source "$_STACK_ENV"
set +a

export OPENCLAW_CONTEXT REPO_ROOT
```

## Step 4: Handle start-browser.sh per-claw domain lookup

`start-browser.sh` constructs a dashboard URL like:

```
https://${OPENCLAW_DASHBOARD_DOMAIN}${OPENCLAW_DASHBOARD_DOMAIN_PATH}/browser/${AGENT}/vnc.html...
```

In the new system, domain is per-claw. The script already identifies the claw via `resolve_gateway` (e.g., `openclaw-personal-claw`). It can then derive the var name:

```bash
GATEWAY=$(resolve_gateway ...) || exit 1
INSTANCE_NAME="${GATEWAY#openclaw-}"
# Convert hyphens to underscores, uppercase for var lookup
CLAW_KEY=$(echo "$INSTANCE_NAME" | tr '-' '_' | tr '[:lower:]' '[:upper:]')
# Dynamic var lookup
DOMAIN_VAR="STACK__CLAWS__${CLAW_KEY}__DOMAIN"
DASHBOARD_PATH_VAR="STACK__CLAWS__${CLAW_KEY}__DASHBOARD_PATH"
CLAW_DOMAIN="${!DOMAIN_VAR}"
CLAW_DASHBOARD_PATH="${!DASHBOARD_PATH_VAR}"
```
