# Plan: Protected Vars + `update-env.mjs`

## Context

Passwords (`ADMINCLAW_PASSWORD`, `OPENCLAW_PASSWORD`) and the AI gateway admin token (`ADMIN_AUTH_TOKEN`) are currently generated ad-hoc by Claude during deployment via `openssl rand` in playbooks. This is brittle:

- Passwords exist only in conversation context — lost when context compresses
- The only record after deployment is buried in `.deploy-logs/` timestamp dirs
- Claude must track generated values through multi-step deploys and output them in the report
- `ADMIN_AUTH_TOKEN` naming is ambiguous (could be any admin token)

**New pattern:** `pre-deploy.mjs` auto-generates all missing protected vars (`.env` > `.env.local` > generate). Playbooks and deploy report read values exclusively from `source-config.sh`. A new `build/update-env.mjs` CLI handles maintenance rotations.

## 2. `build/pre-deploy.mjs` — Generate & Cache Protected Vars

### New functions

- `readEnvLocal()` — reads `.env.local` via `dotenv.parse()`, returns `{}` if missing
- `writeEnvLocal(vars)` — writes key=value pairs to `.env.local` with header comment
- `generateLocalEnv(protectedVars)` — generates `.deploy/local.env` content with `LOCAL__` prefix

### Protected vars registry

```javascript
import { randomBytes } from "crypto";

const PROTECTED_VARS = {
  ADMINCLAW_PASSWORD: () => randomBytes(18).toString("base64"),
  OPENCLAW_PASSWORD: () => randomBytes(18).toString("base64"),
  AI_WORKER_ADMIN_AUTH_TOKEN: () => randomUUID().replace(/-/g, "") + randomUUID().replace(/-/g, ""),
};
```

### In `main()`, after reading `.env`

1. `const envLocal = readEnvLocal()`
2. For each protected var: `env[name] || envLocal[name] || generator()`
3. `writeEnvLocal({ ...envLocal, ...resolved })` — persist for next build
4. After building `.deploy/`, write `.deploy/local.env`

### `.deploy/local.env` output

```bash
# Protected vars — NOT synced to VPS
# Generated by pre-deploy
LOCAL__ADMINCLAW_PASSWORD=MhcL76En+p5W+CL2DIRVNnSf
LOCAL__OPENCLAW_PASSWORD=2XLnQOvgnTH80Y5QcY+NEIiO
LOCAL__AI_WORKER_ADMIN_AUTH_TOKEN=abc123def456...
```

Not synced because `sync-deploy.sh` uses explicit file paths (lines 74-78), not a blanket `.deploy/` rsync.

**Files:** `build/pre-deploy.mjs`

## 3. `deploy/host/source-config.sh` — Source Protected Vars

After sourcing `stack.env`, source `local.env` in local context:

```bash
# Source protected vars (local-only, not synced to VPS)
if [ "$OPENCLAW_CONTEXT" = "local" ]; then
  _LOCAL_ENV="${REPO_ROOT}/.deploy/local.env"
  if [ -f "$_LOCAL_ENV" ]; then
    source "$_LOCAL_ENV"
  fi
fi
```

On VPS: no `local.env` exists → no change. Add `_LOCAL_ENV` to the `unset` line.

**Files:** `deploy/host/source-config.sh`

## 4. `build/update-env.mjs` — Maintenance CLI (new file)

```
node build/update-env.mjs VAR_NAME value         # Set explicit value
node build/update-env.mjs VAR_NAME --generate     # Auto-generate new value
```

Behavior:

- Looks up `VAR_NAME` in the same `PROTECTED_VARS` registry
- Protected → writes to `.env.local`; regular → writes to `.env`
- Updates existing line if present, appends if not
- Prints `VAR_NAME=<value>` and target file to stdout
- `--generate` uses the registered generator (or default hex64 for regular vars)

**Files:** `build/update-env.mjs` (new)

## Verification

1. **Auto-gen first run:** Delete `.env.local`, run `npm run pre-deploy` → `.env.local` created with 3 vars, `.deploy/local.env` has `LOCAL__` prefixed versions
2. **Stability:** Run pre-deploy again → same values (cached in `.env.local`)
3. **Explicit override:** Set `ADMINCLAW_PASSWORD=custom` in `.env` → pre-deploy uses it, `.env.local` gets custom value
4. **source-config.sh:** `source scripts/lib/source-config.sh && echo $LOCAL__ADMINCLAW_PASSWORD` → prints password
5. **Not synced:** `sync-deploy.sh --dry-run` → `local.env` absent from transfer list
6. **update-env.mjs generate:** `node build/update-env.mjs AI_WORKER_ADMIN_AUTH_TOKEN --generate` → writes to `.env.local`, prints value
7. **update-env.mjs regular:** `node build/update-env.mjs LOG_WORKER_TOKEN --generate` → writes to `.env`, prints value
8. **update-env.mjs set:** `node build/update-env.mjs ADMINCLAW_PASSWORD mypass` → writes to `.env.local`
